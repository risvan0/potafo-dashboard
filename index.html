Firebase Configuration
Setup Guide
1. Go to console.firebase.google.com
2. Open project potafo-kam (or create new)
3. Go to Realtime Database → Rules
4. Paste these rules and click Publish:
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
5. Go to Project Settings → General
6. Copy apiKey, projectId, appId, databaseURL
7. Paste them in the fields on the left → Save & Test
🔗 Open Firebase Rules
☁️ Supabase Cloud Setup

Connect your dashboard to a free Supabase database so all KAMs share the same live data across devices.

⚠️ Not connected — using local browser storage only
ℹ️ These credentials are saved in your browser. Each device needs to enter them once.
📋 Setup Guide
STEP-BY-STEP
1. Go to supabase.com → Sign up free
2. Create a new project (choose any region)
3. Go to SQL Editor → run the SQL below
4. Go to Settings → API → copy URL + anon key
5. Paste them above → click Connect
6. Share this HTML file with your team — they paste the same keys
SQL TO RUN IN SUPABASE
create table vendors ( id bigint primary key, name text not null, city text, stage text default 'Onboarding', notes text, created_at text ); create table menu_items ( id bigint primary key, vendor_id bigint references vendors(id) on delete set null, name text not null, rate numeric, from_time text, to_time text, type text default 'Menu', sub_category text, diet text, description text, status text default 'Y', portions text ); create table activities ( id bigint primary key, date text, vendor_id bigint references vendors(id) on delete set null, type text, kam_name text, remarks text ); create table documents ( id bigint primary key, vendor_id bigint references vendors(id) on delete set null, cert_type text, upload_date text, expiry_date text, notes text, file_name text, file_size bigint, file_mime text, status text ); -- Enable public read/write (for internal tool use) alter table vendors enable row level security; alter table menu_items enable row level security; alter table activities enable row level security; alter table documents enable row level security; create policy "allow all" on vendors for all using (true) with check (true); create policy "allow all" on menu_items for all using (true) with check (true); create policy "allow all" on activities for all using (true) with check (true); create policy "allow all" on documents for all using (true) with check (true);
